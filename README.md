AutoService CRM
---

## О проекте

### Почему был создан этот продукт?

AutoService CRM появился как практическая реализация цифровой системы управления автосервисом в формате локального desktop-приложения. Проект объединяет потребности администраторов СТО в контроле клиентской базы, учёте автомобилей и формировании заказ-нарядов с инструментами механиков для выполнения назначенных работ и отслеживания статусов. Разработка велась как комплексный пример полноценного корпоративного приложения с архитектурой MVC, реляционной базой данных и многооконным интерфейсом.

### Какую проблему он решает?

Приложение предоставляет единую цифровую среду для организации работы автосервиса без зависимости от облачных сервисов. Оно автоматизирует рутинные операции: ведение клиентской базы, регистрацию автомобилей, оформление заказ-нарядов, назначение механиков и контроль складских остатков, заменяя разрозненные таблицы и бумажные журналы структурированной системой.

### В чём его ключевые преимущества?

**Полная локальность и безопасность:** Все данные хранятся в локальной базе PostgreSQL на вашем компьютере  
**Ролевая система:** Разграничение прав доступа для администраторов и механиков  
**Комплексное управление СТО:** Клиенты, автомобили, заказ-наряды, услуги и запчасти в одном приложении  
**Автоматизация бизнес-процессов:** Автогенерация номеров нарядов, пересчёт стоимости, триггерный контроль складских остатков  
**Аудит изменений:** История статусов заказ-нарядов и складских движений  
**Готовое рабочее решение:** Полнофункциональное приложение с типовыми сценариями автосервиса «из коробки»

## Технологии

Для работы приложения требуется:

**Java Runtime Environment (JRE) 17 или выше:**
**СУБД PostgreSQL 13 или выше:** 
### Архитектура проекта:

| Компонент | Технология |
|-----------|------------|
| Язык и интерфейс | Kotlin, TornadoFX для современного GUI |
| База данных | PostgreSQL 13+ с полной схемой клиентов, автомобилей, нарядов, услуг и запчастей |
| Архитектура | MVC (Model-View-Controller) с чётким разделением ответственности |
| Дополнительные компоненты | DAO-паттерн для доступа к данным, шифрование паролей (BCrypt), type-safe SQL (Exposed DSL) |

## Возможности

**Система пользователей:** Авторизация с ролями `admin` и `mechanic`, изоляция данных по ролям
**Управление клиентами:** Создание, редактирование и удаление клиентов с указанием ФИО, телефона и контактов
**Управление автомобилями:** Регистрация транспорта с маркой, моделью, VIN, госномером и привязкой к владельцу
**Заказ-наряды:** Оформление нарядов с автогенерацией номеров, выбором клиента, автомобиля и механика
**Статусы нарядов:** Жизненный цикл: Создан → В работе → Ожидание запчастей → Выполнен → Закрыт
**Позиции наряда:** Добавление услуг из справочника и запчастей со склада с автопересчётом стоимости
**Справочник услуг:** Управление перечнем работ, категориями, ценами и нормами часов
**Склад запчастей:** Учёт остатков, контроль критического уровня, приход и расход
**Поставщики:** Регистрация и учёт поставщиков автозапчастей
**История статусов:** Автоматический аудит изменений статусов нарядов
**Складские движения:** Триггерный учёт прихода, расхода и корректировки остатков
**Представления:** Готовые SQL-view для быстрого доступа к данным нарядов и критических остатков

## Использование

### Начало работы

При первом запуске войдите под существующими демо-данными (admin / mechanic) или создайте нового пользователя через БД.

### Главный экран

После входа откроется рабочее пространство с доступом к модулям в зависимости от роли.

### Основные функции (панель инструментов)

| Раздел | Описание |
|--------|----------|
| **Клиенты и автомобили** | Управление клиентской базой и автопарком |
| **Заказ-наряды** | Создание и управление нарядами на ремонт |
| **Склад запчастей** | Учёт остатков и критических запасов |
| **Справочники** | Услуги, поставщики, пользователи |
| **Отчёты** | Выручка, загрузка механиков, статистика |

### Работа с данными

Двойной клик по записи клиента, автомобиля или наряда открывает детальное редактирование
Все изменения сохраняются автоматически через транзакции БД
Фильтрация нарядов по статусу и механику для быстрого доступа
